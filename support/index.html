Bug Reports

Htmlpp is a free product, and well-documented. Before you email us with questions, make sure you read the documentation and especially the FAQ. Bug reports are always welcome, if you can tell us what to do to make the problem happen. When you report a problem, please tell us:

  1. What version of htmlpp you are using
  2. If a code problem, the source code that generates it
  3. The exact error you get
  4. How to reproduce the problem

Htmlpp includes a debugging option that allows you to see what htmlpp is doing with its passes: 

htmlpp -debug filename
This leaves a number of .wrk files lying around; they contain the result of each pass and can help you locate your problem.

Discussion Group

For help about using or extending htmlpp, use the htmlpp discussion group. This is a mailing list kindly hosted by Scott Drake. To subscribe, send a message to htmlpp-request@isonline.com with the following command in the body of the message: subscribe htmlpp.

Registered htmlpp users get automatic news about updates, bugs, and other items of interest. Registration is free. To register, send a message to ebb@imatix.com with register htmlpp as the subject, and additional comments in the body of the text if you wish. We may sometimes use your comments (with permission) on our web site. In any case, if you mail us with any question regarding htmlpp, you are automatically registered.

If you want to change htmlpp, get a hold of the Camel Book (Programming Perl), and the Libero documentation.
